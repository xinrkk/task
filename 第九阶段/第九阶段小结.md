# 第九阶段小结

## 1. 对部署流程有了系统认识

通过本次部署实践，我完整经历了从后端服务运行、Nginx 配置反向代理，到域名解析与访问测试的全过程。相比只在本地运行项目，这次真正理解了“上线”的含义，即让公网用户可以稳定、安全地访问系统。


## 2. 深刻理解了反向代理原理

在配置 Nginx 的过程中，我理解了反向代理的工作机制：客户端请求先到 Nginx，再由 Nginx 转发到后端服务器。这样不仅隐藏了真实服务地址，也让系统结构更加清晰。同时也明白了 Nginx 在整个架构中扮演“统一入口”的重要角色。

## 3. 从“写代码”到“部署系统”的成长

这次实践让我从单纯的后端开发学习，过渡到理解整个服务运行环境。它让我意识到一个完整的系统包含服务器、网络、代理层、应用层等多个部分。今后学习中，我会更加注重整体架构与实际部署能力的提升。
